# Security Policy

## Supported versions

Security fixes are applied to the latest version on the default branch. Older releases are not guaranteed to receive updates.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Report it privately to the repository owner through GitHub's private vulnerability reporting feature. Include a clear description, affected version, reproduction steps, and potential impact.

We aim to acknowledge reports within 7 days and will coordinate a fix and disclosure timeline with the reporter.

## Sensitive data

Never include API keys, authentication headers, book content, database files, diagnostics exports, local paths, or personal information in issues, pull requests, logs, or screenshots. Revoke and rotate any credential that is accidentally exposed before reporting it.

## 安全报告

请勿在公开 issue 中披露疑似安全漏洞。请使用 GitHub 的私密漏洞报告功能联系仓库维护者，并提供受影响版本、复现步骤和影响说明。

请勿在 issue、PR、日志或截图中提交 API 密钥、认证头、书籍正文、数据库、诊断导出、本机路径或个人信息。若凭据意外泄露，请先撤销并轮换该凭据。
